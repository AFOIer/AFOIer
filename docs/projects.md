# OIer 开源项目

> 退役 OIer 们在竞赛之外开发、参与或维护的有趣项目。从 OJ 框架到日常工具，竞赛思维在工程中的延续。

本项目列表通过社区 PR 和 Issue 推荐方式收录。每个人推荐自己使用过或参与过的项目。

## 竞赛相关工具与平台

这些项目直接与算法竞赛生态相关——评测系统、题目管理、训练工具等。

| 项目 | 说明 | 链接 |
|------|------|------|
| **Hydro** | 新一代 OJ 框架，被多个学校和机构采用 | [hydro.js.org](https://hydro.js.org) |
| **Universal OJ (UOJ)** | 由退役 OIer 维护的评测平台，开源 | [github.com/UniversalOJ](https://github.com/UniversalOJ) |
| **LibreOJ (LOJ)** | 开源 OJ，收录大量高质量题目 | [loj.ac](https://loj.ac) |
| **OI-wiki** | 编程竞赛知识整合站点，社区驱动 | [oi-wiki.org](https://oi-wiki.org) |
| **CF-Tool** | Codeforces 命令行工具，支持提交/查看状态等 | [github.com/xalanq/cf-tool](https://github.com/xalanq/cf-tool) |

## OIer 开发的通用项目

算法竞赛选手转型工程后，在各自的领域创造的作品。

> 本板块依赖社区自荐或推荐。如果你（或你认识的 OIer）有值得推荐的开源项目，欢迎通过 PR 或 Issue 添加。

### 收录标准

- 项目主要作者为退役 OIer（有竞赛背景）
- 项目在 GitHub 上开源
- 具有一定的使用价值或影响力
- 项目仍在维护或已形成稳定社区

## 推荐方式

### 通过 PR 添加

1. Fork [AFOIer](https://github.com/AFOIer/AFOIer)
2. 编辑 `docs/projects.md`，按格式添加项目条目
3. 提交 PR，标题以 `[项目]` 开头

### 通过 Issue 推荐

在 [GitHub Issues](https://github.com/AFOIer/AFOIer/issues) 中提交，包含：
- 项目名称
- 项目链接
- 一句话描述
- 作者竞赛背景（可选，如"NOI Au 退役"等）
